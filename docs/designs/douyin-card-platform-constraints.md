# 抖音竖卡平台约束（全局设计规则）

**Date:** 2026-03-13  
**Status:** Design Decision  
**Scope:** 所有抖音竖卡设计、导出脚本、发布指南必须遵守  
**Traceability:** [可传播资产需求](../requirements-planning/spreadable-cognitive-assets-requirements.md) REQ-D-2 / REQ-D-3

---

## 1. 用途

本文件为**竖卡相关的权威约束**。任何竖卡结构调整、新模型发布、导出脚本修改，必须先满足以下规则，再考虑其他设计要素。

---

## 2. 平台约束（必须遵守）

### 2.1 两侧安全边距（Critical）

| 规则 | 说明 | 数值 |
|------|------|------|
| **两侧留白** | 抖音页面左右有工具栏（点赞、评论、分享等），内容必须避开该区域 | 左右各 **≥100px** 安全边距 |
| **实现常量** | `src/app3d/export.js` 中 `DOUYIN_SAFE_MARGIN_H = 100`，与设计保持一致 | 修改需同步更新本文档 |

**原因**：竖卡在抖音全屏展示时，左右两侧被平台 UI 占用；若内容太靠边，会被遮挡或显得拥挤。

**验收**：竖卡有效内容区宽度 = 画布宽度 − 2×安全边距（如 1080 − 200 = 880px）；文字、插图、分隔线不超出该区域。

### 2.2 画布尺寸

| 规则 | 数值 |
|------|------|
| 分辨率 | 1080×1920（9:16） |
| 3 秒可读 | 核心信息在首屏、字号与对比度足够 |

### 2.3 上下边距

| 规则 | 数值 |
|------|------|
| 上下安全边距 | 各 ≥56px（`paddingVertical`） |

## 4. 修改本规则的流程

若平台 UI 变化（如抖音工具栏宽度调整）需变更安全边距：

1. 更新本文件中的数值与说明
2. 更新 `export.js` 中对应常量
3. 重新导出验证图，确认内容不被遮挡
