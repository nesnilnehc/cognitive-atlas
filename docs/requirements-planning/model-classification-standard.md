# 认知模型准入与三维分类标准（MECE）v1.1

## 0. 先后顺序（强制）

本仓库采用两阶段流程，顺序不可颠倒：

1. `阶段 A：模型准入判定`（决定“是否应纳入本仓库”）
2. `阶段 B：三维分类判定`（决定“纳入后位于哪个坐标”）

未通过阶段 A 的候选模型，不进入三维分类。

## 1. 模型准入标准（是否纳入仓库）

### 1.1 准入对象定义

可纳入对象必须是“可复用的认知模型”，即满足：

- 能稳定解决一类思考/分析/决策问题，而非一次性技巧。
- 具备可描述的方法结构（模板/步骤/原则/元模型之一）。
- 可在多个场景重复使用，而非仅适配单一事件。

### 1.2 硬性门槛（全部必过）

- `G1 可复用性`：至少能在 2 个以上独立场景复用。
- `G2 可解释性`：能写出清晰定义、输入、输出和适用边界。
- `G3 可区分性`：与库内已有模型非同义重复；若重叠度高，必须说明差异。
- `G4 可验证性`：至少有 2 个可信来源或公认实践支撑（书籍/论文/权威方法论）。
- `G5 安全合规性`：不直接鼓励违法、歧视、明显不当操控或高风险误用。

任一门槛不通过，结论为“不纳入”。

### 1.3 准入评分（用于优先级，不替代门槛）

在硬性门槛全部通过后，再进行 100 分评分：

- `相关性`（0-25）：与“认知模型库”目标的贴合度。
- `可操作性`（0-20）：用户是否能据此直接行动。
- `迁移性`（0-20）：跨行业/跨问题复用能力。
- `独特性`（0-15）：相对现有模型的新增价值。
- `证据强度`（0-10）：定义与实践证据质量。
- `误用风险`（0-10）：风险越低分越高。

准入建议阈值：

- `>= 75`：建议纳入
- `60-74`：进入观察池（暂缓纳入）
- `< 60`：不纳入

### 1.4 准入输出（最小记录集）

每个候选模型必须记录：

- 模型名（中英文/别名）
- 一句话定义
- 解决问题类型
- 输入/输出
- 适用边界与不适用边界
- 与相近模型差异说明
- 证据来源（至少 2 条）
- 准入结论（纳入/观察/不纳入）

## 2. 目的与范围

本标准用于将每个认知模型唯一映射到三维坐标：

- `X` 时间导向：`-1` 过去 / `0` 现在 / `1` 未来
- `Y` 控制深度：`1` 表达层 / `2` 结构层 / `3` 战略层 / `4` 元认知层
- `Z` 抽象程度：`1` 工具 / `2` 方法 / `3` 原则 / `4` 元模型

适用对象：`model-data.js` 中所有模型，以及后续新增模型。

## 3. MECE 约束

- `Mutually Exclusive（互斥）`：每个模型在每个维度只能有一个主归属值。
- `Collectively Exhaustive（穷尽）`：标准应覆盖全部模型；若模型无法归类，视为标准缺口，进入“修订流程”，不允许临时特判。

## 4. 判定输入（证据卡）

每个模型必须先填写以下证据，再做分类：

- `核心问题`：该模型主要解决什么问题。
- `主要产出`：使用该模型后直接产出的成果形态。
- `典型场景`：在什么决策时点使用（复盘/分析/规划）。
- `操作方式`：模板、步骤法、原则、还是“模型的模型”。

缺少任一项证据，判定无效。

## 5. 三维区域判定标准

### 5.1 X 轴：时间导向（-1/0/1）

- `-1 过去（Retrospective）`
  - 进入标准：主要用于复盘、归因、解释已发生事件。
  - 排除标准：主要价值不在未来行动选择。
- `0 现在（Analysis）`
  - 进入标准：主要用于描述现状、结构化当前问题。
  - 排除标准：不以历史归因或未来规划为主。
- `1 未来（Planning）`
  - 进入标准：主要用于方案选择、优先级、路径规划。
  - 排除标准：不以复盘或现状解释为主。

### 5.2 Y 轴：控制深度（1/2/3/4）

- `1 表达层（Output）`
  - 进入标准：核心目的是“说清楚/写清楚/讲清楚”。
  - 主要产出：叙述、表达结构、沟通文案。
- `2 结构层（Structure）`
  - 进入标准：核心目的是拆解、分类、框架化、穷尽检查。
  - 主要产出：树、矩阵、清单、结构图。
- `3 战略层（Strategy）`
  - 进入标准：核心目的是取舍、排序、资源配置、行动路径。
  - 主要产出：优先级、策略组合、执行路线。
- `4 元认知层（Meta-Cognition）`
  - 进入标准：核心目的是校准思维方式、选择/组合模型、反思认知偏差。
  - 主要产出：思维规则、模型选择规则、认知校准框架。

### 5.3 Z 轴：抽象程度（1/2/3/4）

- `1 工具（Tool）`
  - 进入标准：固定模板或清单，可直接套用，迁移性有限。
- `2 方法（Method）`
  - 进入标准：有明确步骤或流程，可重复执行。
- `3 原则（Principle）`
  - 进入标准：跨场景判断规则，不依赖固定步骤。
- `4 元模型（Meta Model）`
  - 进入标准：用于组织、比较、生成或组合其他模型。

## 6. 标准化判定流程

1. 先判 `Y`（控制深度）。
2. 再判 `Z`（抽象程度）。
3. 最后判 `X`（时间导向）。
4. 每一步都必须给出“进入标准证据 + 排除标准证据”。
5. 形成最终坐标：`(X, Y, Z)`。

说明：按 `Y -> Z -> X` 顺序可减少歧义，因为“模型作用层级”通常比“时间导向”更稳定。

## 7. 冲突裁决规则

当同一维度出现双归属时，按以下优先级裁决：

1. `核心问题优先`：模型最核心回答的问题决定归属。
2. `主要产出优先`：如果核心问题仍并列，以主要产出形态裁决。
3. `典型场景优先`：若仍并列，以最常见应用场景裁决。
4. `强制单值`：最终必须保留一个主值，其他候选写入“备选解释”。

## 8. 质量门槛与审计

- 每个三维判定结果必须包含：
  - `坐标值`：`X/Y/Z`
  - `判定理由`：不超过 120 字
  - `证据卡`：4 项齐全
- 双人独立判定一致率应 `>= 80%`，低于阈值进入复核。
- 所有标准修订需版本化（`v1.1`、`v1.2`），并记录变更原因。
